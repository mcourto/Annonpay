User Journey

  1. The user visits your website and is prompted to log in using their credentials. (This plus the alias would need to be provided to a user, a better option might be to create a one time usage link that has all of this data hidden)
  2. After successful login, the user is presented with a form to input an alias associated with their pre-registered PayID.
  3. The user fills in the alias and submits the form.
  4. The user is then directed to a secure payment page where they enter their payment details (e.g., credit card information) to complete the transaction. (This would actually redirect to stripe otherwise it’s too dodgy looking)
  5. Once the payment is successful, the system retrieves the corresponding PayID from the backend based on the provided alias.
  6. The system automatically purchases TON crypto using the provided funds through a connected crypto exchange (e.g., Binance).
  7. After the TON crypto is purchased, the system initiates a withdrawal, transferring the TON to a hardcoded wallet address in the background.
  8. The TON crypto is received in the hardcoded wallet, completing the transaction.

Benefits for Anonymous Payments

  • Privacy: By using an alias associated with a pre-registered PayID, users can make payments without directly revealing their real identity or PayID, enhancing their privacy.
  • Convenience: Users can easily make payments using their existing banking details and a simple alias, without the need to create additional accounts or go through extensive verification processes.
  • Security: The system utilizes secure payment gateways and encryption to protect user data and prevent unauthorized access to sensitive information. Additionally, the backend holds the PayID, adding an extra layer of security.
  • Automation: The entire process, from payment to TON purchase and transfer, is automated, reducing the need for manual intervention and increasing efficiency.
  • Anonymity: By transferring the purchased TON directly to a hardcoded wallet address in the background, the system maintains the anonymity of the transaction, as the funds are not held in an intermediary account.

Summary

Overall, this system provides a convenient and secure way for users to anonymously purchase TON crypto using their existing banking details and an alias associated with a pre-registered PayID. The backend handling of the PayID adds an extra layer of security and privacy, while the automated process ensures a smooth user experience.
